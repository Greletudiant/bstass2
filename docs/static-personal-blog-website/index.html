Static Personal Blog Website

1 Install Hugo

Hugo is a static site generator that can convert Markdown files into HTML files. Before installing Hugo, you need to install Git, Go, and Dart Sass.

Git:

  • Build Hugo from source
  • Use Hugo module functionality
  • Install themes as Git submodules
  • Access commit information from a local Git repository
  • Host your site using services like CloudCannon, CloudFlare Pages, GitHub Pages, GitLab Pages, and Netlify.

Download links:

Go:

  • Build Hugo from source
  • Use Hugo module functionality

Download link: https://go.dev/doc/install

Dart Sass:

  • Required for using the latest features of the SASS language to convert Sass into CSS.

Download link: https://gohugo.io/functions/css/sass/#dart-sass

Once the above three software packages are installed, you can proceed to install Hugo. It is recommended to install the extended/deploy edition.

Command to install:

CGO_ENABLED=1 go install -tags extended,withdeploy github.com/gohugoio/hugo@latest

2 Quick Start

Open the Hugo official website in your browser: https://gohugo.io/, click on the Themes menu to choose a template you like.

3 Host on GitHub

Host the static website generated locally on GitHub for storage without needing to maintain it yourself. Refer to this article. Update commands are as follows:

(base) ➜  newsite git:(main) ✗ hugo --minify --theme=docura --baseURL="https://Greletudiant.github.io"
(base) ➜  newsite git:(main) cd ./public
(base) ➜  public git:(main) ✗ git add .
(base) ➜  public git:(main) ✗ git commit -m "update"
(base) ➜  public git:(main) ✗ git push origin main

GitHub Pages Static File Deployment

1. Configure GitHub Pages

In the repository settings page, locate the “GitHub Pages” section.
In the “Source” dropdown menu, select “main branch” (or “master branch,” depending on your repository settings).
Click the “Save” button.

2. Access the Website

Wait for a while, and GitHub Pages will automatically build your website.
Visit your website at: https://yourusername.github.io.

Refer to this article for detailed steps.

Netlify for Rendering Static Pages Hosted on GitHub

  1. Netlify Official Website:
    Homepage: https://www.netlify.com/

  2. First, register an account on Netlify. You can log in using GitHub, GitLab, or Bitbucket authorization. After logging in, go to the management center and click “New site from Git” in the top-right corner to add a site.

  3. Based on your hosting platform, choose GitHub, GitLab, or Bitbucket. For demonstration, we will use GitHub (for GitHub users, don’t forget to check the “Limit GitHub access to public repositories” option).

  4. After clicking GitHub, a window will pop up asking for authorization. Click “Authorize netlify,” and Netlify will read all your repositories.

  5. Select the repository you have created, choose the branch (default is master), and click “Deploy site.” The system will automatically compile your static pages and provide a secondary domain for your site.

  6. After creation, wait for a moment. You will see that the free static space on Netlify has been successfully created, and your GitHub code is now running on Netlify.
