Deploy and Utilize Local Large Language Models (LLMs)

Install Ollama

Official installation can be slow. As an alternative, download the compressed package from Baidu Netdisk and install it on your Linux device via SSH. link

scp ollama-linux-amd64.tgz user@10.65.22.149:

Ollama service configuration - Common environment variables link

Download Models Locally

For example, to download the llama3 model:

ollama pull llama3

List local models
ollama list

Start the Ollama service
ollama serve

Check Ollama service status
systemctl status ollama

Interact with the llama3 model in the terminal:

ollama run llama3 "Hello, world!"

Configure Ollama for Browser Access

Configure Ollama to Listen on All Interfaces

  1. Modify the Ollama service configuration:
    • Create a systemd file:
    sudo vim /etc/systemd/system/ollama.service
    • Add the following content:
    [Unit]
Description=Ollama Service
After=network.target

[Service]
User=root
ExecStart=/home/user/bin/ollama serve # your ollama path
Environment="OLLAMA_HOST=0.0.0.0:11434"
Environment=HOME="OLLAMA_ORIGINS=*"
Environment="OLLAMA_MODELS=/data/soft/ollama/.ollama/models"
Restart=always
[Install]
WantedBy=multi-user.target
    • Reload systemd configuration:
    sudo systemctl daemon-reload
    • Start the Ollama service and verify:
    sudo systemctl start ollama
sudo systemctl status ollama # should show active(running)
    Note the model download location for easy transfer from local to server. The default path is /root/.ollama/models. Check with ls -l /root/.ollama/models.
  2. Confirm port listening: 
    netstat -tunlp | grep 11434 # Expected output: tcp6       0      0 :::11434                :::*                    LISTEN      12345/ollama
  3. Verify firewall rules: 
    sudo firewall-cmd --list-ports | grep 11434
    Expected output: 11434/tcp. If not shown, run: 
    sudo firewall-cmd --permanent --add-port=11434/tcp 
sudo firewall-cmd --reload
  4. SElinux troubleshooting: 
    sudo semanage port -l | grep 11434
    If not labeled as http_port_t, run: 
    sudo semanage port -a -t http_port_t -p tcp 11434
  5. Test local browser access: 
    curl http://localhost:11434 # Expected output: Ollama API version info
  6. Test LAN browser access: 
    curl http://<CentOS_IP>:11434

Install Browser Extension

Install the browser extension pageassist, enter http://ip:11434 to interact with the LLM via a web UI.
